# 反馈归因维度参考

在体验反馈类事件中，确保携带足够的归因维度以便后续定位与分析。以下维度为**推荐参考**，根据具体功能按需选用。

## 常用归因维度

| 维度 | 说明 | 适用场景 |
|------|------|---------|
| 页面路径 | 问题发生在哪个页面 | 所有场景 |
| 操作标识 | 用户执行了什么操作 | 用户操作类事件 |
| 错误码 | 具体失败原因 | 失败/异常类事件 |
| 耗时 | 操作或加载花了多久 | 性能相关事件 |
| 请求标识 | 后端请求 ID | 涉及接口调用时 |
| 会话标识 | 串联用户行为链路 | 需要跨事件关联时 |
| 网络类型 | wifi / 4g / 弱网 | 网络敏感场景 |
| 设备型号 | 低端机 vs 高端机 | 性能分群分析 |
| 系统版本 | iOS / Android 版本 | 兼容性排查 |
| 应用版本 | 小程序版本号 | 版本回归对比 |

## 使用原则

1. **按需选用**：不是每个事件都需要全部维度，根据功能和分析诉求选择
2. **业务维度优先**：除通用维度外，优先补充与当前功能直接相关的业务字段
3. **可归因**：确保事件数据能回答"什么条件下出了什么问题"
4. **可聚合**：字段值应使用枚举或有限集合，避免高基数（如不传完整 URL 参数）

## 小程序埋点封装

基础归因参数优先由小程序埋点 API 或项目级 `reportExperienceEvent` 统一补齐，业务代码不要重复手填页面、设备、网络、小程序版本等通用字段。业务侧只按当前功能补充 `action`、`result`、`duration`、`errorCode`、`requestId` 和必要业务维度。
