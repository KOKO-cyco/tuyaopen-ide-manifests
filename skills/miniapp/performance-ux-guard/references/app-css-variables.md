# App CSS Variables（深浅色自适应设计 Token）

> **小程序框架内置变量**：以下变量由框架在 `:root` 与 `:root[data-theme=dark]` 中定义，**所有小程序项目默认可用，无需自行声明**。变量列表稳定（只会增量增加），可放心依赖。
>
> **AI 标准工作流**：拿到设计稿任意 `rgba(...)` / `#xxx` / 字号 / 间距 / 圆角值时，**先来这里反查匹配变量**，找到就用 `var(--app-XX)`，不要直接写字面量。

## TL;DR

| 设计稿写法 | ❌ 别这么写 | ✅ 这么写 |
|---|---|---|
| 主标题 `rgba(0,0,0,0.90)` | `color: rgba(0,0,0,0.90)` | `color: var(--app-B6-N1)` |
| 卡片底 `rgba(255,255,255,1)` | `background: #fff` | `background: var(--app-B6)` |
| 蒙层 `rgba(0,0,0,0.20)` | `background: rgba(0,0,0,0.20)` | `background: var(--app-B6-N6)` |
| 圆角 12px | `border-radius: 12px` | `border-radius: var(--app-C3_2)` |
| 16px 正文 | `font-size: 16px; line-height: 24px` | `font-size: var(--app-T6-f); line-height: var(--app-T6-h)` |

## rgba → 变量名 反查表（AI 拿到设计稿先查这里）

> **怎么用**：拿到设计稿的 `rgba()` 值，按"颜色类型"找下面对应小节，匹配透明度，再按上下文挑具体变量。

### 浅底上的黑色叠加（文本 / 图标 / 描边 / 蒙层）

| `rgba(0, 0, 0, 0.70)` | N2 / N5 | `--app-B*-N2` 或 `--app-B*-N5`（语义不同，dark mode 下值不同） |
| `rgba(0, 0, 0, 0.50)` | N3 | `--app-B*-N3` / `--app-B*_2-N3` |
| `rgba(0, 0, 0, 0.40)` | N8 | `--app-B*-N8` / `--app-B*_2-N8` |
| `rgba(0, 0, 0, 0.30)` | N4 | `--app-B*-N4` / `--app-B*_2-N4` |
| `rgba(0, 0, 0, 0.20)` | N6 | `--app-B*-N6` / `--app-B*_2-N6` |
| `rgba(0, 0, 0, 0.10)` | N7 | `--app-B*-N7` / `--app-B*_2-N7` |

## 命名约定

### 类别前缀

| 前缀 | 含义 | 用途 |
|---|---|---|
| `B1` ~ `B6` | Background（中性背景，白/浅灰基底） | 页面背景 / 卡片底色 |
| `B*_2` | Background 变体（灰阶基底） | 二级容器 / 分隔区 |
| `B_Login` / `B_Activator` | 场景专用背景 | 登录页 / 设备激活页等独立场景 |
| `M1` ~ `M5` | Main（品牌主色） | `M1` 主蓝 / `M2` 警示红 / `M3` 成功绿 / `M4` 副蓝 / `M5` 链接蓝 |
| `M*_1` | Main 浅化变体（0.20 透明度） | 品牌色弱化用法，如 chip / tag 背景 |
| `M*_2` | Main 加深/亮度变体 | 按下态 / hover 强调 |
| `T1` ~ `T13` | Typography 字号体系 | 配 `-f` / `-p` / `-h` 三套尺寸 |
| `P0` ~ `P9` | Padding/间距 0-40px | 内/外边距 |
| `I1` ~ `I7` | Icon 尺寸 16-64px | 图标本身大小 |
| `IC1` ~ `IC6` | Icon Container 容器尺寸 | 图标点击区/包围盒 |
| `C1_*` ~ `C3_*` | Corner/圆角 | 不同级别容器圆角 |

### 透明度后缀 `-N1` ~ `-N9`

适用于 `B*` / `M*` 系列，表示**在父级基底上叠加的前景色**：

| 后缀 | 透明度 | 在 `B*`（浅底）上 | 在 `M*`（品牌底）上 |
|---|---|---|---|
| N1 | 0.90 | 主文本（近纯黑） | 反白主文本（近纯白） |
| N2 | 0.70 | 次要文本 | 次要反白文本 |
| N3 | 0.50 | 弱化文本 / 占位文字 | 弱化反白文本 |
| N4 | 0.30 | 描边 / 分隔线 | 反白描边 |
| N5 | 0.70 | 备选语义（同 N2 透明度，dark mode 下与 N2 不同） | 反白备选 |
| N6 | 0.20 | 浅蒙层 / 浅描边 | 反白浅蒙层 |
| N7 | 0.10 | 极浅蒙层 / 极浅描边 | 反白极浅 |
| N8 | 0.40 | icon / 边框中等 | 反白中等 |
| N9 | 0.03 | 极淡填充 | 反白极淡 |

**核心机制**：

- 父级是 `B*`（浅基底）→ `-NX` 是**黑色叠加** `rgba(0,0,0,X)`
- 父级是 `M*`（品牌色）→ `-NX` 是**白色叠加** `rgba(255,255,255,X)`
- dark mode 下变量定义在 `:root[data-theme=dark]` 中自动翻转，**业务代码无感**

### Typography 后缀 `-f` / `-p` / `-h`

| 后缀 | 含义 | 示例 |
|---|---|---|
| `-f` | font-size | `--app-T6-f: 16px` |
| `-p` | letter-spacing 或 paragraph-spacing | `--app-T6-p: 4px` |
| `-h` | line-height | `--app-T6-h: 24px` |

## 实操示例（完整还原一张卡片）

设计稿描述：白底卡片，圆角 12px，内边距 16px；标题 18px 黑字 90%；描述 12px 灰字 50%；底部蒙层 20% 黑。

| 描述文字 | 12px, `rgba(0,0,0,0.50)` | T2 + B6-N3 | `font-size: var(--app-T2-f); line-height: var(--app-T2-h); color: var(--app-B6-N3)` |

## 何时**不该**用 `--app-*`

- 设计稿明确标注"不随主题切换"的固定色（如品牌 logo 死值、第三方 SDK 提供的固定色）
- 第三方 widget / iframe 内部样式（CSS 变量无法穿透 iframe）
- 临时调试 / 一次性 demo 样式
- 设计稿出现**反查表中没有的 rgba 值**：先与设计同学确认是否设计稿用错色（大概率是），再考虑特殊处理

## 注意事项

- **同一 rgba 值可能对应多个变量**（如 `--app-B6-N6`、`--app-B3-N6` 都是 `rgba(0,0,0,0.20)`），按**容器上下文就近原则**挑选，review 起来语义最清晰。
- 这套变量是**小程序框架沉淀**，所有项目默认可用；变量列表**只会增量增加，不会重命名/删除**，可放心依赖。本表反映的是当前框架版本，新增变量后会增量更新。
- Ray 小程序在 H5 渲染容器内可直接使用 CSS 变量。极少数运行容器不支持 CSS 变量时需提前评估降级方案。
