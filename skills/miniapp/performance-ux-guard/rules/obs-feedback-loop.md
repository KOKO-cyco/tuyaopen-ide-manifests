# 体验治理闭环：准入、召回、度量

## Rule
体验治理必须形成「准入阶段 → 召回阶段 → 度量闭环」的完整流程。

## Bad
```text
流程：上线前人工检查 → 线上问题仅靠用户投诉发现
```
被动且滞后，无法持续发现问题。

## Good
```text
准入：上线前自动化体验检查 gate
召回：线上指标异常自动报警 + 归因分析
度量：指标看板跟踪治理效果
```
形成检查、发现、验证的完整链路。

## Why
单点检查无法持续保障体验。闭环流程确保问题不仅被发现，还能被定位、修复并验证效果。
