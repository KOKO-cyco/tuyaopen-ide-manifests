# smart-ui 版本对齐规则

## 什么时候读这个文件

- 方案中使用了 smart-ui 组件
- 用户提到版本差异、组件不存在、props 不兼容

## 检查顺序

1. 读取项目 `package.json`
2. 查看是否存在 `@ray-js/smart-ui`
3. 若存在，对照 smart-ui skill 中当前要使用的组件文档确认版本是否支持
4. 若版本明显不匹配，明确提示用户升级或改用当前版本可用方案

## 处理原则

- 不要默认文档中的所有组件和 props 都能在当前项目版本使用
- 本 Skill 不提供本地 smart-ui 组件文档，相关能力必须以 smart-ui skill 文档为准
- 如果无法确认版本能力，优先保守回答
- 版本不匹配时，不要继续输出依赖该能力的确定性代码

## 输出建议

- 说明是否检查了 `package.json`
- 说明当前方案是否依赖某个 smart-ui 能力
- 若有风险，明确写出“需要用户确认或升级”
