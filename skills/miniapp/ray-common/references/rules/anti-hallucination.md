# 防幻觉规则

## 基本原则

- 没查到，不等于存在
- 语义相似，不等于 API 相同
- 截图像某个组件，不等于它支持那个 props

## 必须遵守

- 禁止虚构 Ray 生命周期
- 禁止虚构 Ray API 名称、参数、返回值、回调字段
- 禁止虚构业务组件或 smart-ui 组件 props、事件、插槽能力
- 禁止把猜测写成确定事实

## 未命中时的处理方式

- 明确说明“未在文档中找到”
- 如有必要，给出相邻目录或备选路径
- 若必须继续产出，标记为“经验推断”，并保持最小假设

## 典型高风险场景

- 图片转代码
- 用户只给组件语义，不给具体组件名
- 升级迁移后接口变动
- 看起来像 React / 微信小程序写法，但用户实际要的是 Ray
