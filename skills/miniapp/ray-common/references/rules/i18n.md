# i18n 规则

## 最高优先级规则

- 业务代码中禁止硬编码中文或其他直接文案
- 统一使用：
  - `import Strings from '@/i18n'`
  - `Strings.getLang('key')`

## 禁止项

- 禁止在业务组件中直接引入 `@ray-js/panel-sdk` 的 I18N 类
- 禁止在页面或组件里直接写中文按钮文案、标题、提示语

## 推荐做法

- key 使用语义化英文命名，如 `confirmButtonText`、`networkErrorTitle`
- 新页面和新组件默认把所有文案都纳入 i18n
- 图片转代码时，视觉稿里的文案只作为语义来源，不直接写死到代码中

## 生成前检查

- 所有用户可见文案是否都走 `Strings.getLang`
- 是否还有硬编码中文
- 文案 key 是否表达语义而不是位置
