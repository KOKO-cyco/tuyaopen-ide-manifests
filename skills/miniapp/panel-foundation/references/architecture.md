# Architecture — Tuya Ray Panel MiniApp

> All paths below are relative to the project's `source/miniapp/` directory
> (the root where `project.tuya.json` lives). All examples are from the actual
> TuyaOpen-IDE-generated panel template; reproduce them in your project to
> verify match.

## 1. 物理目录结构

```
source/miniapp/
├── project.tuya.json         ← Tuya 平台项目元数据（不进代码逻辑）
├── ray.config.ts             ← Ray CLI 构建配置（webpack alias、plugins）
├── package.json              ← 依赖：@ray-js/{ray, panel-sdk, cli, smart-ui} + lodash-es
├── tsconfig.json
├── cdn/                      ← 静态资源（CDN 路径根，build 时上传）
├── typings/                  ← Tuya 平台 Kit 类型定义（BaseKit/MiniKit/DeviceKit/…）
├── scripts/                  ← 项目构建脚本（patch-webpack.js 等）
└── src/                      ← 业务代码
    ├── app.config.ts         ← 全局窗口配置 + 页面列表
    ├── app.tsx               ← App 启动逻辑（class App + initPanelEnvironment + SdmProvider）
    ├── app.less              ← 全局样式（最小化使用）
    ├── ty-shim.ts            ← TuyaOpen IDE 预览 shim（合成 window.ty + gzlJSBridge）
    ├── routes.config.ts      ← 路由表（与 app.config.ts pages 配套）
    ├── global.config.ts      ← 全局配置常量
    ├── devices/              ← 设备模型（SmartDeviceModel + dpKit）
    │   ├── index.ts          ← SmartDeviceModel + dpKit 实例、离线预览逻辑
    │   └── schema.ts         ← DP 定义（IDE 自动生成，不要手改）
    ├── config/
    │   └── dpCodes.ts        ← DP code 常量（可选，类型安全）
    ├── pages/                ← 页面
    │   └── home/             ← 每个页面一个目录
    │       ├── index.tsx
    │       ├── index.config.ts
    │       ├── index.module.less
    │       └── components/
    │           └── <ComponentName>/
    │               ├── index.tsx
    │               └── index.module.less
    └── utils/                ← 通用工具
```

**注意**：`dp.config.json` 在旧版模板中存在，现已从模板移除。
`src/devices/schema.ts` 是唯一的 DP schema 来源，由 IDE "Sync DPs from Cloud" 直接生成。

## 2. 配置文件职责

### 2.1 `project.tuya.json` —— Tuya 平台项目元数据

⚠️ **这个文件 IDE 维护，但有几个字段你必须懂**：

```jsonc
{
  "name": "tuyaopen-miniapp",
  "type": "panel-app",                    // 必须是 panel-app（区别于通用小程序）
  "ownerType": "tuya-official",           // tuya-official | tuya-thirdparty
  "appid": "",                            // ⚠️ 上传前必填（Tuya 平台创建 panel 后获得）
  "projectId": "",                        // Tuya 平台 project id
  "appVersion": "1.0.0",                  // semver，每次发版递增
  "i18n": false,                          // ⚠️ 生产建议改为 true
  "publicRoot": "./cdn",                  // CDN 资源根目录
  "miniprogramRoot": "./dist/tuya",       // 构建产物输出
  "dependencies": {                       // Tuya Kit 依赖版本（影响审核）
    "BaseKit": "3.0.0",
    "MiniKit": "3.1.0",
    "DeviceKit": "4.6.1",
    "BizKit": "4.5.1",
    "HomeKit": "3.1.4"
  },
  "baseversion": "2.30.18",               // 引擎版本
  "devMode": "ray",                       // 必须是 ray
  "compileType": "miniprogram"            // 必须是 miniprogram（面板属于这种）
}
```

**最常变更的字段**：`appid`（首次创建后填）、`appVersion`（每次发版）、
`dependencies.*`（升级 Tuya Kit 时）。

### 2.2 `app.config.ts` —— 全局窗口 + 页面列表

```ts
export const thing = {
  window: {
    backgroundColor: '#f2f4f6',
    navigationBarTitleText: '',
    navigationBarBackgroundColor: '#f2f4f6',
    navigationBarTextStyle: 'black',
  },
  pages: ['pages/home/index'],            // ⚠️ 每个页面必须在这里登记
};
```

新建页面时：在 `src/pages/<name>/index.tsx` 写组件 + 把
`pages/<name>/index` 加进 `pages` 数组。**没登记的页面无法路由到**。

### 2.3 `src/devices/schema.ts` —— DP schema（IDE 直接生成）

```ts
import { keyBy } from 'lodash-es';
import { GetSmartDeviceModelDpSchema } from '@ray-js/panel-sdk';

// Auto-generated by TuyaOpen IDE — do not edit manually.
// Regenerate via the MiniApp panel's "Sync DPs from Cloud" action.
export const lampSchema = [
  {
    attr: 0, canTrigger: true, code: 'switch', defaultRecommend: true,
    editPermission: false, executable: true, id: 1, mode: 'rw', name: 'Switch',
    property: { type: 'bool' },
    type: 'obj',
  },
  // … more DPs
] as const;

export const lampSchemaMap = keyBy(lampSchema, 'code') as GetSmartDeviceModelDpSchema<typeof lampSchema>;
```

**作用**：
- `lampSchema` —— 完整 DP 列表（`const` tuple），给 TypeScript 提供精确类型推导
- `lampSchemaMap` —— 以 code 为 key 的索引对象，供 `devices/index.ts` 中 `createDpKit` 使用

⚠️ **此文件由 IDE「Sync DPs from Cloud」按钮整体替换**，不要手改。
要加新 DP → 先在 Tuya 平台产品里加 → IDE 同步。

### 2.4 `src/devices/index.ts` —— 设备模型 + dpKit + 离线预览

```ts
import { SmartDeviceModel } from '@ray-js/panel-sdk';
import { createDpKit } from '@ray-js/panel-sdk/lib/sdm/interceptors/dp-kit';
import { getLaunchOptionsSync } from '@ray-js/ray';
import { lampSchema, lampSchemaMap } from './schema';

type SmartDeviceSchema = typeof lampSchema;

// dpKit 是协议桥，将原始 DP 事件翻译为 SmartDeviceModel 的格式。
// 在 app.tsx 的 onLaunch 里通过 dpKit.init(device) 初始化。
export const dpKit = createDpKit<SmartDeviceSchema>({ protocols: {} });

export const devices = {
  common: new SmartDeviceModel<SmartDeviceSchema>({
    interceptors: dpKit.interceptors,
  }),
};
```

**关键函数**（同文件导出）：
- `isOfflinePreview()` —— 返回是否在 IDE 预览中（无真实设备）
- `initOfflinePreview()` —— 预填 SmartDeviceModel 状态 + 重定向 publishDps 到本地回环
- `installOfflineBridge()` —— 安装 `tymp` postMessage 桥与 IDE chassis 通信

**生产代码通常不需要直接调用这些函数**；它们在 `app.tsx` 的顶层已经初始化好了。

### 2.5 `src/ty-shim.ts` —— TuyaOpen IDE 预览 shim

此文件在 IDE 预览 iframe 里运行（无真实 `gzlJSBridge`），合成 `window.ty` 和
`window.gzlJSBridge` 两个对象，把所有面板 API 调用通过 `postMessage` 路由给
父 webview。它是 IDE 专属 shim，**不影响生产上线**（真实 Tuya App 里已有原生桥）。

⚠️ `ty-shim.ts` 内部使用了 `localStorage` 作为存储回退，仅供预览。这是唯一允许
在 `src/` 内使用 `localStorage` 的文件，不适用 [conventions.md Rule 10](conventions.md#rule-10)。

### 2.6 `ray.config.ts` —— 构建配置

```ts
const config = {
  resolveAlias: {
    '@pages': path.resolve(__dirname, 'src', 'pages'),
  },
  plugins: [
    {
      name: 'fix-web-target',
      configWebpack({ config }) {
        config.resolve.alias.merge({
          '@pages': path.resolve(__dirname, 'src', 'pages'),
        });
      },
    },
  ],
};
module.exports = config;
```

`@pages` 别名让 route loader 的 `import('@pages/home/index')` 能正确解析。

## 3. App 启动流程

```
ray dev / ray build
    ↓
src/app.tsx 被框架加载
    ↓
import './ty-shim'              ← IDE 预览 shim（真机上此文件无副作用）
import './app.less'             ← 全局样式
    ↓
initPanelEnvironment({          ← @ray-js/ray 提供
  useDefaultOffline: true        ← 离线预览支持
})
    ↓
initOfflinePreview()            ← 离线时预填 SmartDeviceModel 并安装 tymp 桥
                                   真实设备时此函数立即 return，不影响生产路径
    ↓
class App extends React.Component
  onLaunch() {
    if (!devices.common.initialized) {
      devices.common.init();
      devices.common.onInitialized(device => dpKit.init(device));
    } else {
      dpKit.init(devices.common);   // 离线预览已初始化
    }
  }
    ↓
render() {
  return <SdmProvider value={devices.common}>{children}</SdmProvider>
}
    ↓
useProps / useActions hooks 在 SdmProvider 内才生效
```

**关键 import**：

| 包 | 用途 |
|---|---|
| `@ray-js/ray` | 框架核心（initPanelEnvironment、View、Text、navigate*、getLaunchOptionsSync） |
| `@ray-js/panel-sdk` | 设备模型（SmartDeviceModel、SdmProvider、useProps、useActions、createDpKit） |
| `@ray-js/smart-ui` | Tuya 官方 UI 组件库（按需引入） |
| `@ray-js/cli` | 构建工具（ray dev / ray build） |
| `lodash-es` | `keyBy` 等工具（schema map 构建） |
| `@reduxjs/toolkit` + `react-redux` | 可选状态管理（跨组件全局 UI 状态） |
| `@tuya-miniapp/cloud-api` | 云端 API 调用（按需，不是默认依赖） |

## 4. DP 数据流（最核心的模型）

```
┌──────────────────────────────────────────────────────────────────┐
│  Tuya 平台（云端）                                                 │
│  dpId / code / type / mode / property 在此定义                    │
└────────────────────────────┬─────────────────────────────────────┘
                             │ IDE "Sync DPs from Cloud" 按钮
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  src/devices/schema.ts                                            │
│  - lampSchema（DP 完整列表，const tuple，TypeScript 真值源）       │
│  - lampSchemaMap（code → schema item 索引，供 createDpKit 使用）  │
└────────────────────────────┬─────────────────────────────────────┘
                             │ build 进 bundle
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  运行时：SmartDeviceModel（在 src/devices/index.ts 创建）          │
│  - dpKit 桥接原始 DP 事件 → model state                           │
│  - 接收 MQTT 消息 → 解析为 DP 变化                                │
│  - 暴露给 useProps / useActions                                   │
└──────────┬──────────────────────────────────────┬────────────────┘
           │ 读                                   │ 写
           ▼                                      ▼
┌────────────────────────────┐  ┌────────────────────────────────┐
│ const props = useProps()   │  │ const actions = useActions()   │
│ const v = props.switch     │  │ actions.switch.set(true)       │
│                            │  │   ↓ publishDps 到 MQTT          │
│ → 自动响应式更新             │  │   → 云端 → 设备 → 设备回执      │
└────────────────────────────┘  └────────────────────────────────┘
```

**关键事实**：
- `useProps(p => p.<code>)` 拿到的是**响应式**的 DP 当前值，DP 一变 UI 就重渲
- `useActions().<code>.set(value)` 触发**异步**下发，返回 `Promise<boolean>`
- **没有 `useState`** 来管 DP —— state 在 SmartDeviceModel 里，你只是订阅它
- `useState` **可以用于本地 UI 状态**：输入 draft、进度条、错误提示 —— 不涉及 DP 的纯前端状态

## 5. `ty` 全局对象

面板运行时提供全局 `ty` 对象用于系统级 API（toast / loading / storage / 路由等）。
在 TypeScript 中声明：

```ts
declare const ty: any;
```

常用 API：

| API | 用途 |
|---|---|
| `ty.showToast({ title, icon })` | 提示条（success / error / loading / none） |
| `ty.showLoading({ title })` / `ty.hideLoading()` | 全屏 loading |
| `ty.showModal({ title, content, showCancel, ... })` | 确认弹窗 |
| `ty.showActionSheet({ itemList, ... })` | 操作菜单 |
| `ty.getStorage({ key, success })` | 持久化读（异步） |
| `ty.setStorage({ key, data })` | 持久化写（异步） |
| `ty.getStorageSync(key)` | 持久化读（同步） |
| `ty.getSystemInfo({ success })` | 系统信息（平台 / 屏幕尺寸） |
| `ty.getSystemInfoSync()` | 系统信息（同步） |
| `ty.getDeviceInfo({ success })` / `ty.getDevInfo({ success })` | 当前设备信息（在线状态 / 名称 / PID） |
| `ty.publishDps({ dps, ... })` | 直接下发 DP（绕过 SmartDeviceModel，慎用） |
| `ty.checkOtaUpgrade()` | 检查固件升级（返回 Promise） |
| `ty.downloadOtaUpgrade({ onProgress })` | 下载升级包 |
| `ty.startOtaUpgrade()` | 执行升级 |
| `ty.openFunctionalSettings()` | 打开设置页（关于 / 定时 / 更多） |
| `ty.openOfficialPanel({ panelType })` | 打开平台内置面板（`'privacy'` 等） |
| `ty.navigateBack()` | 返回上一页 |
| `ty.scanCode({ success })` | 二维码扫描 |

⚠️ `ty.*` 是 Tuya 平台原生 API，不是 `wx.*` / `tt.*`。在 TuyaOpen IDE 预览中，
`ty-shim.ts` 会合成 `ty` 对象并通过 postMessage 路由调用，行为与真机保持一致。

## 7. 样式约定（CSS Modules）

⚠️ **强制要求**：所有页面 / 组件用 `.module.less`，不要用全局 `.less`。

```less
// pages/home/index.module.less
.hero {
  background: #fff;
  &.heroOn { color: green; }
  &.heroOff { color: gray; }
}
```

```tsx
// pages/home/index.tsx
import styles from './index.module.less';
<View className={`${styles.hero} ${value ? styles.heroOn : styles.heroOff}`} />
```

**为什么必须 `.module.less`**：
- 类名自动命名空间化，不污染全局
- 多个面板组合时不互相覆盖样式
- raypack v1.7.x 已知 bug：`?modules.js` 查询参数模式不可靠，必须用文件
  扩展名 `.module.less` 才能正确生成 hash 类名

## 8. 离线预览机制

TuyaOpen IDE webview 里的面板预览不携带真实 `deviceId`，SmartDeviceModel
无法 `init()`。`initOfflinePreview()` 通过以下方式解决：

1. 从 `lampSchema` 推导各 DP 的默认值，预填 model 的内部状态
2. 将 `initialized` 置为 `true`，让 `SdmProvider` 正常渲染子树
3. 重定向 `publishDps` 到本地回环（actions 能驱动 UI 更新）
4. 通过 `tymp` postMessage 协议与 IDE chassis 同步 DP 事件

**判断逻辑**：

```ts
function isOfflinePreview(): boolean {
  const launch: any = getLaunchOptionsSync() || {};
  const { deviceId, groupId } = launch.query || {};
  return !(deviceId || groupId);
}
```

业务代码通常不需要关心这个分支，但 **在 `useActions().X.set()` 失败时**，
检查是不是在离线态（离线下 cloud API 调用会失败）。
